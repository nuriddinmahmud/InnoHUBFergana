# Compiler Backend API Guide

To enable real-time code compilation and execution, the frontend has been configured to communicate with a backend API. This document outlines the expected API endpoint that the backend needs to expose.

## Code Execution Endpoint

The backend should provide an endpoint that accepts code, the programming language, and returns the execution result.

| Method | Path | Body | Success Response | Error Response |
|--------|------|------|----------|----------|
| POST | `/compiler/run` | `{ "language": "python", "code": "print('hello')" }` | `{ "output": "hello\n", "error": "" }` | `{ "output": "", "error": "SyntaxError: invalid syntax" }` |

### Request Body

- `language` (string): The programming language of the code to be executed. The frontend will send one of the following values: `"javascript"`, `"python"`, `"cpp"`, `"dart"`.
- `code` (string): The source code to be executed.

### Response Body

- `output` (string): The standard output (stdout) generated by the executed code.
- `error` (string): The standard error (stderr) generated by the executed code. If there are no errors, this should be an empty string.

## Backend Implementation Notes

- **Security**: Code execution must be handled in a secure, sandboxed environment (e.g., using Docker containers) to prevent malicious code from affecting the server.
- **Languages**: The backend should be prepared to handle the four specified languages.
- **Endpoint**: The endpoint should be exposed under the same base URL as the other APIs (e.g., `http://localhost:3000/api/compiler/run`).

Once this endpoint is implemented and available, the compiler feature in the web application will function with real-time code execution.
